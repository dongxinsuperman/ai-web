# 安全策略

## 支持范围

安全修复优先面向当前 `main` 分支。旧提交、个人部署的环境配置和第三方站点本身不在本仓库的修复范围内。

## 报告漏洞

请**不要**在公开 Issue、讨论区、截图或 Pull Request 中披露漏洞细节、可利用请求、真实 token、Cookie 或账号信息。

仓库公开前，维护者需要在 GitHub 仓库设置中启用 **Private vulnerability reporting**。启用后，请使用仓库的私密漏洞报告入口提交；若该入口暂不可用，请通过仓库维护者的 GitHub 主页私信联系，并只提供最小复现信息。

报告应尽量包含：

- 受影响的版本或 commit；
- 漏洞影响与前置条件；
- 可脱敏的复现步骤；
- 建议修复方向（可选）。

## 响应方式

维护者会确认问题、评估影响、安排修复，并在修复发布后再公开必要信息。请给维护者合理的修复时间，避免在修复前公开披露。

## 部署者责任

AI Web 会处理浏览器执行、站点登录态、回调与报告。部署者必须保护 `AIWEB_API_TOKEN`、模型 Key、数据库、Cookie、登录配方和报告目录；详见 [开源边界与安全](docs/开源边界与安全.md)。
