# AI Web 对外接口文档（供工作台 / CI 集成）

> 面向把 AI Web 当作**执行器**接入的调用方（如 case-flow 工作台）。
> 设计目标：与 **ai-phone** 对外契约**尽量同构**，调用方接 AI Web 时改动最小。
>
> 核心差异（被 AI Web 这边吸收）：
> - ai-phone 调度真机（android/ios/harmony）；**AI Web 调度浏览器**。
> - **`platform` = 浏览器类型**：支持 `chrome`(Chromium)、`firefox`、`webkit`/`safari`（Safari 引擎）。
>   该字段**真实生效**——调用方传哪个，AI Web 就用对应真实浏览器引擎执行（基于 Playwright）。
> - AI Web 没有真实"设备"，但**像 ai-phone 一样"几个端有几台"**：每个引擎单独配台数
>   （如 `chrome:2,firefox:1,webkit:1`），投影成浏览器槽 `chrome-1`、`chrome-2`、`firefox-1`、`webkit-1`，
>   暴露与 ai-phone 同形的设备端点。**该引擎台数 = 它最多同时执行几个任务**（每引擎独立并发；
>   总并发 = 各台数之和）。同引擎内各槽等价（调度器任挑空闲）；**跨引擎是真实不同的浏览器**。
>   台数由 `AIWEB_BROWSER_SLOTS` 控制，也可在维护台热改；某引擎台数=0 即不启用（提交会被拒）。
>   需先 `playwright install` 对应浏览器。

基址（本地默认）：`http://127.0.0.1:8009`；报告等静态资源在 `/files/...`（返回的报告 URL 已是绝对地址）。

---

## 1. 鉴权

内网匿名，不带 token（与 ai-phone 一致）。若部署设置了 `AIWEB_API_TOKEN`，则所有 `/api` 需带
`Authorization: Bearer <token>`；回调由 AI Web 主动发出，无需调用方鉴权。

---

## 2. 浏览器（设备）就绪查询

为与 ai-phone 统一，AI Web 暴露同形端点：按**每个引擎各自配置的台数**投影成浏览器槽。
例如配置 `chrome:2,firefox:1,webkit:1`，则返回 4 个槽：`chrome-1`、`chrome-2`、`firefox-1`、`webkit-1`。
忙/闲按**该引擎**当前执行中数量标记。

```http
GET /api/devices/available     # 空闲浏览器槽（可接单）
GET /api/devices/statuses      # 全部浏览器槽（含忙/闲）
```

`/api/devices/statuses` 响应（字段与 ai-phone 等价，便于复用）：

```json
[
  {
    "serial": "chrome-1",
    "alias": "Chrome #1",
    "platform": "chrome",
    "brand": "Chromium",
    "model": "Chromium",
    "osVersion": "",
    "screenWidth": 1280,
    "screenHeight": 800,
    "status": "online",
    "effectiveStatus": "idle",
    "lock": null
  }
]
```

- 每个引擎的槽位数量 = 该引擎配置的台数；`effectiveStatus` 为 `busy`/`idle` 按**该引擎**执行中数量标记。
- `/api/devices/available` 只返回 `idle` 槽，字段更精简。
- **同一引擎内选哪个槽对执行无影响**（各槽等价）；**跨引擎是真实不同的浏览器**，按 `platform` 选。

## 3. 提交执行

```http
POST /api/submissions
Content-Type: application/json
```
```json
{
  "submissionName": "回归冒烟-2026-06-27",
  "callbackUrl": "{调用方}/callback/{token}",
  "retryMax": 0,
  "cacheMode": "off",
  "functionMapContext": "可选：登录规则 / 测试账号 / 业务术语 / 异常处理等只读执行参考",
  "items": [
    {
      "caseId": "cf-123",
      "caseName": "登录后进入个人中心",
      "runContent": "测试标题/前置条件/操作步骤/预期结果 拼接文本",
      "platforms": ["chrome", "firefox"],
      "deviceAliasPools": {"chrome": ["chrome-1"]},
      "assets": ["avatar.png"]
    }
  ]
}
```

字段说明：

| 字段 | 必填 | 说明 |
|---|---|---|
| `submissionName` | 否 | 批次名；缺省回落 `submissionId` |
| `callbackUrl` | 否 | 终态回调地址；只支持 http(s)；best-effort、不重试 |
| `retryMax` | 否 | 失败重试上限，默认 0 |
| `cacheMode` | 否 | **接受但忽略**（轨迹缓存暂未实现，恒按 off 处理；传了不会报错） |
| `functionMapContext` | 否 | **支持**：只读执行参考，注入 system prompt，帮助 AI 更稳执行；不改变 runContent 的任务范围 |
| `items` | 是 | 非空数组 |
| `items[].caseId` | 是 | 调用方业务键（如 `cf-<id>`）；批次内唯一 |
| `items[].caseName` | 否 | 展示名 |
| `items[].runContent` | 是 | 自然语言执行目标 |
| `items[].platforms` | 否 | **浏览器引擎**数组，默认 `["chrome"]`；可选 `chrome`/`firefox`/`webkit`(或 `safari`)。**多端 fan-out（对齐 ai-phone）：勾几个端，这条 case 就在每个端各跑一次**，生成多个 item（同 `caseId`、不同 `platform`）。任一引擎台数=0（未启用）则**整批拒绝**（`rejectReason=unsupported_platform`） |
| `items[].deviceAliasPools` | 否 | `{浏览器类型:[槽别名]}`；**收下但忽略**（同引擎各槽等价，调度器任挑空闲） |
| `items[].assets` | 否 | 引用 AI Web 素材库中的文件名（执行中 upload_file 用） |

响应：

```json
{
  "submissionId": "7f1a2b3c4d5e",
  "submissionName": "回归冒烟-2026-06-27",
  "acceptedAt": "2026-06-27T09:00:00+08:00",
  "items": [
    {"itemId": "abc123", "caseId": "cf-123", "platform": "chrome", "state": "queued"}
  ]
}
```
> 返回必含 `submissionId`。每个 item 带 `platform`（浏览器类型）。

准入失败 `400`：`{"detail": {"rejectReason": "...", "rejectDetail": "...", "index": 0}}`，
常见 `rejectReason`：`invalid_body` / `missing_field`（缺 caseId/runContent）/ `duplicate_case_id` / `unknown_asset`。

## 4. 查询

```http
GET /api/submissions/{submissionId}
GET /api/submissions/{submissionId}/items/{caseId}?include_run=true&platform=firefox
```
批次返回 `state` / `counts` / `summaryReportUrl` / `items[]`（每条含 `platform`、`state`、`reportUrl`）。
单条 `include_run=true` 附带 run + 每步（动作/思考/截图 URL/耗时）。
**多端时一个 `caseId` 对应多条 item**，用 `?platform=` 精确定位某一端（不传取第一条）。
取消同理：`POST .../cases/{caseId}/cancel?platform=firefox` 只取消该端，不传则取消该 case 全部端。

---

## 5. 取消

```http
POST /api/submissions/{submissionId}/cancel                    # 整批
POST /api/submissions/{submissionId}/cases/{caseId}/cancel     # 单条
```
queued → 立即 `cancelled`；running → 标记取消，worker 在检查点中止。

## 6. 回调（AI Web → 调用方）

事件名与 ai-phone 一致；调用方按 `event=="submission.item.terminal" 或 含 caseId` 判定单条事件，否则批次事件。

单条终态 `submission.item.terminal`：

```json
{
  "event": "submission.item.terminal",
  "version": 1,
  "submissionId": "7f1a2b3c4d5e",
  "submissionName": "回归冒烟-2026-06-27",
  "itemId": "abc123",
  "caseId": "cf-123",
  "caseName": "登录后进入个人中心",
  "platform": "chrome",
  "engine": "web-vlm",
  "state": "success",
  "statusReason": "run_success",
  "runId": "9abc123",
  "attempts": 1,
  "elapsedMs": 62000,
  "steps": 8,
  "tokenStats": {"total_tokens": 12345, "cached_tokens": 8000},
  "reportUrl": "http://127.0.0.1:8009/files/reports/7f1a2b3c4d5e/cf-123.html"
}
```
> `platform`（浏览器类型）+ `caseId` 供调用方按 `(caseId, platform)` 匹配 item。
> `state`：`success` / `failed`；`statusReason` 含 `needs_human`（模型请求人工）等归因。

批次终态 `submission.terminal`：

```json
{
  "event": "submission.terminal",
  "version": 1,
  "submissionId": "7f1a2b3c4d5e",
  "submissionState": "done",
  "counts": {"success": 2, "failed": 1},
  "summaryReportUrl": "http://127.0.0.1:8009/files/reports/7f1a2b3c4d5e/_summary.html"
}
```

## 7. 与 ai-phone 契约差异对照

| 维度 | ai-phone | AI Web |
|---|---|---|
| 执行对象 | 真机 android/ios/harmony | 浏览器（每引擎配台数的槽） |
| `platform` | 操作系统平台 | **浏览器引擎**（chrome/firefox/webkit，真实生效） |
| 设备端点 | 真实设备列表 | **每引擎各自台数投影成浏览器槽**，同形返回 |
| 并发模型 | 每台真机=1 并发 | **每引擎独立台数**（几个端有几台），总并发=各台数之和 |
| `deviceAliasPools` | 真实别名池，影响派发 | **收下即忽略**（同引擎各槽等价） |
| `cacheMode` | 轨迹缓存 v1/v2/v3 | **收下即忽略**（暂未实现） |
| `functionMapContext` | 支持 | **支持** |
| `assets` | 无 | 支持（素材库） |
| 提交/查询/取消/回调/匿名 | — | **完全同构** |
