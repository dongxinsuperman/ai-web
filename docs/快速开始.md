# 快速开始

本指南让你在一台机器上跑通最小链路：**PostgreSQL + AI Web Server + Browser Agent + Chromium**。

> 即使 Server 和 Agent 在同一台机器，它们也必须是两个进程。只启动 Server，任务会进入队列，但不会有人真正打开浏览器执行。

## 1. 前置条件

- Python 3.11+
- Node.js 20+（仅维护台需要）
- 一个可访问的 PostgreSQL 数据库
- 豆包、Claude 或 OpenAI 兼容的视觉模型配置

创建一个空数据库，例如 `aiweb`。首次启动 Server 会自动创建所需表。

## 2. 安装 Server 与 Agent 依赖

```bash
git clone https://github.com/dongxinsuperman/aiweb.git
cd aiweb/backend

python3.11 -m venv .venv
source .venv/bin/activate
pip install -e ".[agent]"
cp .env.example .env
```

编辑 `backend/.env`。最少需要确认：

```dotenv
AIWEB_DATABASE_URL=postgresql+asyncpg://<user>:<password>@127.0.0.1:5432/aiweb
AIWEB_VLM_PROVIDER=doubao
AIWEB_VLM_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
AIWEB_VLM_API_KEY=<你的真实 Key>
AIWEB_VLM_MODEL=<你的模型或 endpoint>
AIWEB_BROWSER_SLOTS={"mac-01":{"chrome":1,"firefox":0,"webkit":0}}
```

`AIWEB_BROWSER_SLOTS` 的节点名必须和 Agent 的 `--agent-id` 一致。上例表示 `mac-01` 可同时运行一个 Chromium 任务。

## 3. 启动 Server

```bash
cd aiweb/backend
source .venv/bin/activate
python -m aiweb.main
```

默认监听 `http://127.0.0.1:8009`。可用以下命令确认：

```bash
curl http://127.0.0.1:8009/health
```

## 4. 安装浏览器并启动 Agent

另开一个终端：

```bash
cd aiweb/backend
source .venv/bin/activate
python -m playwright install chromium
python -m aiweb.agent --server http://127.0.0.1:8009 --agent-id mac-01
```

需要 Firefox 或 WebKit 时，再安装对应浏览器：

```bash
python -m playwright install firefox webkit
```

Server 的维护台或接口 `GET /api/browser-agents` 能看到 Agent 后，才表示执行节点已就绪。

## 5. 投递第一条任务

```bash
curl -X POST http://127.0.0.1:8009/api/submissions \
  -H 'Content-Type: application/json' \
  -d '{
    "submissionName": "hello-web",
    "items": [{
      "caseId": "example-title",
      "runContent": "打开 https://example.com 并验证标题包含 Example",
      "platforms": ["chrome"]
    }]
  }'
```

响应中的 `submissionId` 可用于查询进度：

```bash
curl http://127.0.0.1:8009/api/submissions/<submissionId>
```

任务结束后，响应会包含 `reportUrl` 或 `summaryReportUrl`。报告中有每步截图、模型思考、动作和最终结果。

## 6. 启动维护台（可选）

```bash
cd aiweb/web
npm install
npm run dev
```

访问终端输出的地址，默认是 `http://127.0.0.1:8010`。维护台用于查看批次、素材、站点免登和 Agent 容量配置。

## 常见的第一处问题

如果任务一直处于 `queued`，先检查两件事：

1. `AIWEB_BROWSER_SLOTS` 中是否有与 Agent ID 相同的节点名；
2. Agent 是否仍连接到同一个 Server。

这不是模型问题，而是“没有空闲执行节点”。完整问题清单见 [故障排查](故障排查.md)。
